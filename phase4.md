# Phase 4 — Basic Turn-by-Turn Drone Simulation

## Level

Intermediate

## Main Objective

Implement a basic simulation engine that moves all drones from the start hub to the end hub through one valid path generated by your Phase 3 pathfinding system.

At the end of this phase, your program must simulate drone movements turn by turn and print them using the exact format required by the Fly-in subject.

This phase focuses on:

* Drone state management
* Simultaneous movement
* Turn-by-turn simulation
* Zone occupancy
* Waiting
* Correct movement output
* Detecting invalid or deadlocked simulations

You will use only **one shared path** during this phase.

Multiple-path allocation, connection capacities, restricted-zone multi-turn movement, and advanced scheduling will be handled in later phases.

# 4. Drone Model

Create a `Drone` class or dataclass representing one drone in the simulation.

A recommended model is:

```python
from dataclasses import dataclass


@dataclass
class Drone:
    """Represent one drone moving through an assigned path."""

    identifier: int
    path_index: int = 0
    delivered: bool = False
```

## Attribute Meanings

### `identifier`

The unique number of the drone.

Examples:

```text
1
2
3
```

These identifiers are printed as:

```text
D1
D2
D3
```

### `path_index`

The index of the drone’s current position inside the shared path.

For this path:

```python
["start", "a", "b", "end"]
```

The indexes are:

```text
0 -> start
1 -> a
2 -> b
3 -> end
```

A new drone begins with:

```python
path_index = 0
```

### `delivered`

Indicates whether the drone has reached the end hub.

Initially:

```python
delivered = False
```

After reaching the end:

```python
delivered = True
```

Delivered drones must no longer participate in the simulation.

# 8. Turn-Based Simulation

The simulation runs in discrete turns.

During each turn:

1. Inspect every active drone.
2. Determine whether it can move to the next zone.
3. Respect zone capacities.
4. Apply valid movements.
5. Record the movement output.
6. Mark drones reaching the end as delivered.
7. Continue until every drone is delivered.

A drone may move at most one path step during one turn.

For this phase, every movement costs exactly one turn.

Restricted-zone two-turn movement is not part of Phase 4.

---

# 9. Shared Path

All drones use the same path.

Example:

```python
path = ["start", "a", "b", "end"]
```

Every drone must follow:

```text
start -> a -> b -> end
```

A drone may not:

* Skip a zone
* Move backward
* Switch to another path
* Move twice during one turn
* Enter an occupied full zone

---

# 10. Zone Capacity Rules

You must respect the `max_drones` value of intermediate zones.

By default:

```text
max_drones = 1
```

A zone with:

```text
max_drones=2
```

may contain up to two drones simultaneously.

## Start Hub

The start hub has unlimited capacity.

All drones begin in the start hub.

Do not prevent multiple drones from remaining in the start hub.

## End Hub

The end hub also has unlimited capacity.

Any number of drones may arrive at the end during the same turn.

Once a drone reaches the end:

* Mark it as delivered
* Remove it from occupancy tracking
* Do not process it in later turns

## Intermediate Zones

Every intermediate zone must respect its configured capacity.

For example:

```text
hub: a 2 3
```

has default capacity:

```text
1
```

This means only one drone can occupy `a` after a turn is completed.

---

# 11. Simultaneous Movement

Drone movement must be evaluated as simultaneous.

This is one of the most important requirements in this phase.

Consider:

```text
start -> a -> b -> end
```

At the beginning of a turn:

```text
D1 is in b
D2 is in a
D3 is in start
```

All intermediate zones have capacity `1`.

The following movements should be allowed during the same turn:

```text
D1: b -> end
D2: a -> b
D3: start -> a
```

The output should be:

```text
D1-end D2-b D3-a
```

This is valid because:

* `D1` leaves `b`
* `D2` enters the newly freed `b`
* `D2` leaves `a`
* `D3` enters the newly freed `a`

A naive algorithm may incorrectly prevent `D2` from entering `b` because `D1` occupied it at the start of the turn.

Your engine must account for zones being freed by drones moving out during the same turn.

---

# 12. Drone Processing Order

Because all drones share one path, process drones closest to the end first.

For example, sort active drones using:

```python
active_drones = sorted(
    active_drones,
    key=lambda drone: (-drone.path_index, drone.identifier),
)
```

This means:

* Drones farther along the path move first
* When two drones have the same position, the lower identifier moves first

Example order:

```text
D1 at b
D2 at a
D3 at start
```

Processing order:

```text
D1
D2
D3
```

This prevents a drone behind another drone from being processed before the front drone has had a chance to free the next zone.

# 13. Occupancy Tracking

At the beginning of every turn, create an occupancy structure.

You may use:

```python
dict[str, int]
```

or:

```python
collections.Counter[str]
```

Example:

```python
{
    "a": 1,
    "b": 1,
}
```

Do not include:

* Delivered drones
* Drones in the start hub
* Drones in the end hub

The start and end hubs are not limited by normal occupancy rules.

A recommended occupancy calculation is conceptually:

```python
for drone in drones:
    if drone.delivered:
        continue

    current_zone = path[drone.path_index]

    if current_zone == start_name:
        continue

    if current_zone == end_name:
        continue

    occupancy[current_zone] += 1
```

# 14. Applying a Movement

When a drone moves:

1. Find its current zone.
2. Find its next zone.
3. Decrease occupancy of the current intermediate zone.
4. Increase occupancy of the destination intermediate zone.
5. Increase the drone’s `path_index`.
6. Mark it delivered if it reached the end.
7. Add its movement to the current turn output.

Example:

```text
D2 moves from a to b
```

State changes:

```python
occupancy["a"] -= 1
occupancy["b"] += 1
drone.path_index += 1
```

Output movement:

```text
D2-b
```

When moving from the start:

* Do not decrease start occupancy

When moving to the end:

* Do not increase end occupancy
* Mark the drone as delivered

---

# 15. Waiting

A drone must stay in place when its next zone has no available capacity.

Example:

```text
start -> a -> b -> end
```

Current state:

```text
D1 is in a
D2 is in start
```

Suppose `D1` cannot move into `b`.

Then `D2` cannot enter `a`, because `D1` remains there.

`D2` must wait in the start hub.

Waiting drones must not appear in the movement output.

Example:

```text
D1-b
```

If `D2` stayed in place, do not print:

```text
D2-start
```

and do not print:

```text
D2-wait
```

Only actual movements are printed.

---

# 16. Required Output Format

Each turn must be represented by one line.

Each movement follows:

```text
D<ID>-<destination_zone>
```

Examples:

```text
D1-a
```

```text
D1-b D2-a
```

```text
D1-end D2-b D3-a
```

Movement entries on one turn are separated by a single space.

Do not include:

* Commas
* Brackets
* Python lists
* Arrows
* Extra descriptions in official output mode

Invalid:

```text
[D1-a, D2-b]
```

Invalid:

```text
D1 -> a
```

Invalid:

```text
Drone 1 moved to a
```

Valid:

```text
D1-a D2-b
```

# 17. Movement Output Ordering

For predictable output, sort movements by drone identifier before printing them.

Your internal processing order may be based on path position, but the final output should preferably be ordered like:

```text
D1-end D2-b D3-a
```

not:

```text
D3-a D1-end D2-b
```

You can store movements as:

```python
list[tuple[int, str]]
```

Example:

```python
[
    (3, "D3-a"),
    (1, "D1-end"),
    (2, "D2-b"),
]
```

Then sort:

```python
movements.sort(key=lambda movement: movement[0])
```

And join:

```python
line = " ".join(text for _, text in movements)
```

# 18. Simulation Loop

The simulation continues until every drone is delivered.

Conceptually:

```python
turns: list[str] = []

while not all(drone.delivered for drone in self.drones):
    movements = self.run_turn()

    if not movements:
        raise SimulationError("simulation is deadlocked")

    movements.sort(key=lambda movement: movement[0])

    line = " ".join(text for _, text in movements)
    turns.append(line)

return turns
```

The caller can then print the result:

```python
for line in simulation.run():
    print(line)
```

---

# 19. Deadlock Detection

A deadlock happens when:

* At least one drone is not delivered
* No drone can move during the current turn

Without deadlock detection, your simulation could create an infinite loop.

Example:

```python
while not all_delivered:
    run_turn()
```

If no drone can move, this loop would run forever.

Instead, detect:

```python
if not movements:
    raise SimulationError("simulation is deadlocked")
```

This protects your program from:

* Invalid paths
* Impossible capacities
* Logic bugs
* Unexpected map states

The error should be handled gracefully in `main.py`.

Example:

```text
Error: simulation is deadlocked
```

Do not show a Python traceback during normal execution.

# 20. Suggested Simulation Algorithm

Use the following algorithm for every turn.

## Step 1: Build Current Occupancy

Count all active drones currently inside intermediate zones.

## Step 2: Select Active Drones

Exclude delivered drones.

## Step 3: Sort Active Drones

Sort them by:

1. Highest `path_index`
2. Lowest drone identifier

## Step 4: Evaluate Each Drone

For each drone:

1. Find the next zone.
2. Check whether the destination is the end.
3. If it is the end, allow movement.
4. Otherwise, inspect destination capacity.
5. If capacity is available, move.
6. Otherwise, leave the drone in place.

## Step 5: Update Occupancy Immediately

When a move is approved:

* Free the source zone
* Reserve the destination zone immediately

This allows later drones in the same turn to use newly freed zones while preventing two drones from reserving the same final capacity.

## Step 6: Return Movements

Return all successful movement records for that turn.

# 25. Main Program Integration

Update `main.py` so the complete flow becomes:

```text
1. Read command-line arguments
2. Parse the map
3. Build the graph
4. Find one path
5. Create the simulation
6. Run the simulation
7. Print every turn
```

Conceptually:

```python
def main() -> int:
    try:
        map_data = MapParser().parse_file(map_path)

        pathfinder = Pathfinder(map_data)
        path = pathfinder.find_shortest_path()

        simulation = Simulation(map_data, path)
        turns = simulation.run()

        for turn in turns:
            print(turn)

        return 0

    except ParseError as error:
        print(f"Parsing error: {error}")
        return 1

    except PathNotFoundError as error:
        print(f"Pathfinding error: {error}")
        return 1

    except SimulationError as error:
        print(f"Simulation error: {error}")
        return 1
```

Do not place all simulation logic directly in `main.py`.

`main.py` should coordinate the application, not implement the algorithm.

# 34. Complexity Analysis

You must be able to explain the complexity during peer evaluation.

Let:

* `D` be the number of drones
* `P` be the path length
* `T` be the total number of simulation turns

## Initialization

Creating all drones:

```text
O(D)
```

## Occupancy Calculation Per Turn

Inspecting every drone:

```text
O(D)
```

## Sorting Active Drones Per Turn

Sorting drones:

```text
O(D log D)
```

## Movement Processing Per Turn

Evaluating each drone:

```text
O(D)
```

## Total Simulation Complexity

Approximately:

```text
O(T × D log D)
```

If drones remain permanently sorted by position or are processed using a suitable queue, the implementation may later be optimized.

For Phase 4, `O(T × D log D)` is acceptable and easy to reason about.

## Memory Complexity

Drone storage:

```text
O(D)
```

Path storage:

```text
O(P)
```

Occupancy storage:

```text
O(P)
```

Turn output storage, when all output lines are retained:

```text
O(total movements)
```

# 35. Important Edge Cases

Your solution must handle all of these.

## Zero Intermediate Zones

```text
start -> end
```

All drones arrive in one turn.

## One Drone

The simulation must work normally without special-case failures.

## Many Drones

The algorithm must not create one hardcoded variable per drone.

## Capacity Greater Than Drone Count

If a zone capacity is larger than the total number of drones, all drones may enter when otherwise allowed.

## Duplicate Coordinates

Zone names must be unique, but different zones may potentially share coordinates unless your parser requirements explicitly reject that.

Simulation logic must use zone names, not coordinates, as identity.

## Blocked Zone

A blocked zone must never appear in a simulation path.

## Delivered Drones

Delivered drones must not be processed again.

## Waiting Drones

Waiting is valid and must not generate output.

## No Movement

If drones remain and no movement occurs, raise a deadlock error instead of looping forever.

# 36. Common Mistakes to Avoid

## Mistake 1: Moving Drones From Start to End Immediately

Wrong:

```python
for drone in drones:
    drone.delivered = True
```

Drones must follow every zone in the path.

## Mistake 2: Processing Drones From Back to Front

Processing a drone in the start before a drone in the next zone may cause unnecessary waiting.

Process drones closest to the end first.

## Mistake 3: Using Only the Occupancy at the Start of the Turn

This prevents valid pipeline movement.

Update occupancy as movements are approved.

## Mistake 4: Counting the Start Hub

All drones share the start hub, so it must not be treated as capacity `1`.

## Mistake 5: Counting the End Hub

The end hub has unlimited capacity, and delivered drones are no longer tracked.

## Mistake 6: Printing Waiting Drones

Only movements are printed.

## Mistake 7: Mutating a List During Iteration

Do not remove drones from the main drone list while iterating over it.

Use the `delivered` attribute instead.

## Mistake 8: No Deadlock Check

A `while` loop without deadlock detection can run forever.

## Mistake 9: Returning Only Printed Text From Internal Logic

Keep simulation logic structured.

Prefer:

```python
list[tuple[int, str]]
```

inside `run_turn`, then format it in `run`.

## Mistake 10: Mixing Simulation and Pathfinding

The pathfinder chooses a route.

The simulation moves drones through the chosen route.

Keep these responsibilities separated.

# 38. Completion Checklist

Phase 4 is complete only when all the following are true.

## Drone State

* [ ] Every drone has a unique identifier.
* [ ] Every drone stores its current path index.
* [ ] Every drone has a delivered state.
* [ ] Drone identifiers begin at `1`.

## Simulation

* [ ] All drones start at path index `0`.
* [ ] Every drone follows the same Phase 3 path.
* [ ] A drone moves at most one zone per turn.
* [ ] Drones closest to the end are processed first.
* [ ] Waiting drones remain in their current positions.
* [ ] Delivered drones are not processed again.
* [ ] The simulation stops only when all drones are delivered.

## Capacity

* [ ] Start capacity is unlimited.
* [ ] End capacity is unlimited.
* [ ] Intermediate-zone capacity is respected.
* [ ] `max_drones` values greater than one work correctly.
* [ ] Moving out frees capacity during the same turn.
* [ ] Two drones cannot exceed a destination’s capacity.

## Output

* [ ] Every movement uses `D<ID>-<zone>`.
* [ ] Movements are separated by one space.
* [ ] Waiting drones are omitted.
* [ ] Each turn appears on a separate line.
* [ ] Movements are ordered by drone identifier.
* [ ] No extra text appears in official output mode.

## Validation

* [ ] Empty paths are rejected.
* [ ] Paths with the wrong start are rejected.
* [ ] Paths with the wrong end are rejected.
* [ ] Unknown zones are rejected.
* [ ] Blocked zones are rejected.
* [ ] Missing connections are rejected.
* [ ] Deadlocks raise a clear error.

## Quality

* [ ] All functions have type hints.
* [ ] All public classes and methods have docstrings.
* [ ] `flake8 .` passes.
* [ ] Required `mypy` checks pass.
* [ ] Unit tests pass.
